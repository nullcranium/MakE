You are a senior Android software engineer with strong experience in offline-first application design.
Your responsibility is to design and implement an Android application for a small catering business (PO System).

# PRODUCT CONTEXT
- **Business Model:** Pre-Order (PO) Catering.
- **Workflow:** Orders are collected over several days (e.g., Mon-Tue) for a specific **Delivery Date** (e.g., Wed).
- **Volume:** Small scale (~100 items/batch).
- **Users:**
  1. Kasir (Record orders, assign to specific Delivery Date)
  2. Dapur (View production totals for a specific Delivery Date)

# CORE REQUIREMENTS
1. **Settings / Configuration**
   - Business Name.
   - Menu Management: Name, Price, Active/Inactive. (Editable).
   
2. **Order Entry (Kasir)**
   - **Delivery Date Selection** (Critical: Assigns order to a batch).
   - Customer Name.
   - Menu Selection: **Simple Dropdown** (Valid for <10 items).
   - Quantity (Default 1).
   - Notes (Free text).
   - Price: Auto-filled, snapshot at time of order.

3. **Order Management**
   - **Filter by Delivery Date.** (Default to nearest upcoming date).
   - Edit orders (change quantity/notes) before the delivery date.
   - Delete/Cancel orders.

4. **Dapur Mode (Kitchen Preview)**
   - Filter by **Delivery Date**.
   - Aggregate totals by Menu for that date.
   - Show: "Menu Name - Total Qty".
   - Expand to show details: Customer Names + Notes.
   - **Filter:** By keyword in notes (e.g., "pedas").

5. **PDF Export**
   - Generate PDF for the selected **Delivery Date**.
   - Content: Grouped List (Menu, Qty, Notes, Customer).
   - Action: Share via Intent (WhatsApp/Bluetooth).

# DATA MODEL
**Entity: Menu**
- id, name, price, is_active

**Entity: Order**
- id
- customer_name
- created_at (Timestamp)
- **delivery_date** (Timestamp/DateString - CRITICAL for PO grouping)
- status (Active/Cancelled)

**Entity: OrderItem**
- order_id, menu_id, quantity, notes, price_snapshot

# ARCHITECTURE
- **Language:** Kotlin
- **Framework:** Android Native (Jetpack Compose or View System - Engineer's Choice for speed/stability).
- **Architecture:** MVVM + Clean Architecture principles.
- **Database:** Room (SQLite).
- **DI:** Manual or Hilt (Keep it simple).

# CONSTRAINTS
- Offline Only.
- No Over-engineering.
- Android Only.
